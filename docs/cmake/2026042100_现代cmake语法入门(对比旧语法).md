# Modern CMake Quickstart（含旧语法对照）

***本文由AI生成。***

这份文档给出常用的 CMake 基础写法，重点是 **现代 CMake（target-based）** 的推荐方式，并附带常见旧写法对照。

---

## 1. 核心理念

- 现代 CMake：围绕 `target`（可执行文件/库）组织配置。
- 旧式 CMake：大量全局变量和目录级命令，容易产生隐式影响。
- 推荐原则：
  - 优先使用 `target_*` 系列命令。
  - 依赖通过 `target_link_libraries` 传递，不手写全局 include/link 参数。
  - 三方库优先 `find_package` + imported targets。

---

## 2. 最小工程骨架

### 推荐（现代）

```cmake
cmake_minimum_required(VERSION 3.20)
project(MyApp LANGUAGES CXX)

add_executable(my_app src/main.cpp)
target_compile_features(my_app PRIVATE cxx_std_20)
```

### 旧写法（可用但不推荐）

```cmake
cmake_minimum_required(VERSION 3.5)
project(MyApp)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
add_executable(my_app src/main.cpp)
```

说明：`CMAKE_CXX_STANDARD` 是全局变量；`target_compile_features` 是 target 级别，更精确。

## 3. 源文件与头文件组织

### 推荐（现代）

```cmake
add_library(core STATIC
    src/core.cpp
    include/core/core.hpp
)

target_include_directories(core
    PUBLIC
        ${CMAKE_CURRENT_SOURCE_DIR}/include
)
```

### 旧写法

```cmake
include_directories(${CMAKE_CURRENT_SOURCE_DIR}/include)
add_library(core STATIC src/core.cpp)
```

说明：`include_directories()` 是目录级全局影响，容易污染其他 target。

---

## 4. 编译选项与宏定义

### 推荐（现代）

```cmake
target_compile_options(core PRIVATE /W4)
target_compile_definitions(core PUBLIC CORE_USE_FAST_PATH=1)
```

### 旧写法

```cmake
add_compile_options(/W4)
add_definitions(-DCORE_USE_FAST_PATH=1)
```

说明：全局编译选项可能影响第三方代码或所有子目录 target。

---

## 5. 依赖链接与可见性（PUBLIC/PRIVATE/INTERFACE）

```cmake
add_library(utils STATIC src/utils.cpp)
target_link_libraries(utils PUBLIC core)
```

- `PRIVATE`：仅当前 target 使用。
- `PUBLIC`：当前 target 使用，且向依赖它的下游传递。
- `INTERFACE`：当前 target 不编译源码，仅向下游传递使用要求。

旧习惯常见问题：直接 `link_libraries(...)` 或手写 linker flags，依赖边界不清晰。

---

## 6. 三方库：`find_package` 优先

### 推荐（现代）

```cmake
find_package(GTest CONFIG REQUIRED)
target_link_libraries(my_tests PRIVATE GTest::gtest_main)
```

### 旧式/兜底

```cmake
find_library(GTEST_LIB gtest PATHS /path/to/lib)
find_path(GTEST_INCLUDE gtest/gtest.h PATHS /path/to/include)
```

说明：  
有 `*Config.cmake` 时优先 `find_package`；只有裸头文件和库文件时再用 `find_library/find_path`。

## 7. 测试（CTest + GoogleTest）

### 推荐（现代）

```cmake
enable_testing()
add_executable(unit_tests tests/sample_test.cpp)
target_link_libraries(unit_tests PRIVATE core GTest::gtest_main)

include(GoogleTest)
gtest_discover_tests(unit_tests)
```

常用命令（多配置生成器如 VS）：

```bash
ctest --test-dir build -N -C Debug
ctest --test-dir build -C Debug -R SampleTest.BasicArithmetic --output-on-failure
ctest --test-dir build -C Debug --output-on-failure
```

## 8. 生成文件（configure_file）

### 推荐

```cmake
set(GENERATED_DIR ${CMAKE_CURRENT_BINARY_DIR}/generated)
configure_file(config.h.in ${GENERATED_DIR}/config.h @ONLY)
target_include_directories(core PUBLIC ${GENERATED_DIR})
```

要点：生成文件应放在构建目录（binary dir），不要写回源码目录。

## 9. 安装与导出（基础）

```cmake
install(TARGETS core my_app
    RUNTIME DESTINATION bin
    LIBRARY DESTINATION lib
    ARCHIVE DESTINATION lib
)
```

若做可复用库，建议进一步学习 `install(EXPORT ...)` 与 package config 导出。

## 10. 常见“旧 -> 新”速查

- `include_directories(...)` -> `target_include_directories(...)`
- `add_definitions(...)` -> `target_compile_definitions(...)`
- `add_compile_options(...)` -> `target_compile_options(...)`
- `set(CMAKE_CXX_STANDARD ...)` -> `target_compile_features(... cxx_std_XX)`
- `link_libraries(...)` -> `target_link_libraries(...)`
- 手工拼库路径 -> `find_package(...)` + imported target

## 11. 一个简洁的推荐模板

```cmake
cmake_minimum_required(VERSION 3.20)
project(MyProject LANGUAGES CXX)

add_library(core STATIC src/core.cpp)
target_compile_features(core PUBLIC cxx_std_20)
target_include_directories(core PUBLIC ${CMAKE_CURRENT_SOURCE_DIR}/include)

add_executable(app src/main.cpp)
target_link_libraries(app PRIVATE core)

find_package(GTest CONFIG QUIET)
if(GTest_FOUND)
    enable_testing()
    add_executable(unit_tests tests/sample_test.cpp)
    target_link_libraries(unit_tests PRIVATE core GTest::gtest_main)
    include(GoogleTest)
    gtest_discover_tests(unit_tests)
endif()
```

## 12. 迁移建议（从旧工程到现代）

- 先从“新增 target 用现代写法”开始，不必一次性重构全项目。
- 优先替换全局命令：`include_directories/add_definitions/link_libraries`。
- 三方库优先切换到 `find_package` 的 imported targets。
- 每迁移一块就本地构建和跑测试，避免大改后难排错。
